一、Model（模型层，保存项目数据）

1. Project
功能：
- 整个项目的核心类。
- 管理所有 Task、Dependency、Resource、Assignment。
- 提供增加、删除、查询等操作。
- 负责项目整体的数据管理。

2. Task（抽象基类）
功能：
- 表示一个任务。
- 保存所有任务共有的信息，如名称、工期、开始时间、结束时间等。
- 作为所有任务类型的父类。

3. BasicTask（普通任务）
功能：
- 继承自 Task。
- 表示普通任务，没有额外特殊功能。

4. MilestoneTask（里程碑任务）
功能：
- 继承自 Task。
- 表示里程碑任务，与普通任务区别在于持续时间等规则。

5. Dependency
功能：
- 表示两个任务之间的依赖关系。
- 记录前驱任务和后继任务。
- 用于关键路径计算、拓扑排序等。

6. Resource
功能：
- 表示项目中的资源。
- 可以是人员、设备、材料等。
- 保存资源自身的信息。

7. Assignment
功能：
- 表示 Task 与 Resource 的关联关系。
- 一个 Task 可以对应多个 Resource。
- 一个 Resource 也可以分配给多个 Task。
- 保存一次资源分配的信息（如数量、工时等）。


二、Importer（导入模块）

8. Importer（抽象基类）
功能：
- 定义导入接口。
- 负责将文件读取为 Project 对象。

9. PPMImporter
功能：
- 继承自 Importer。
- 实现 PPM 格式文件的导入。


三、Exporter（导出模块）

10. Exporter（抽象基类）
功能：
- 定义导出接口。
- 负责将 Project 保存到文件。

11. PPMExporter
功能：
- 继承自 Exporter。
- 实现 PPM 格式文件的导出。


四、Controller（控制层）

12. Controller（单例）
功能：
- 接收用户操作。
- 调用 Project 完成业务逻辑。
- 调用 Importer、Exporter 完成文件读写。
- 协调 View 与 Model。


五、View（界面层）

13. ConsoleView
功能：
- 显示菜单。
- 获取用户输入。
- 输出操作结果。
- 不直接修改任何项目数据。
